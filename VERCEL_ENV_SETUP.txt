[VERCEL ENVIRONMENT VARIABLES SETUP]

Repo đã chuyển sang Next.js full-stack tại `empulse-next`.
Hãy vào Vercel Dashboard -> Settings -> Environment Variables và thêm các biến sau:

--- CORE (BẮT BUỘC) ---
1. DATABASE_URL
   Giá trị: postgresql://postgres.xxxx:[PASSWORD]@aws-0-xxx.pooler.supabase.com:6543/postgres?pgbouncer=true
   (Dùng connection string Transaction Mode nếu có)

2. DIRECT_URL
   Giá trị: postgresql://postgres.xxxx:[PASSWORD]@aws-0-xxx.pooler.supabase.com:5432/postgres
   (Bắt buộc cho Prisma migrations)

3. JWT_SECRET
   Giá trị: Chuỗi ngẫu nhiên đủ dài (ví dụ: my-super-secret-key-2026)

--- OPTIONAL (NẾU DÙNG SUPABASE CLIENT) ---
4. NEXT_PUBLIC_SUPABASE_URL
   Giá trị: https://xxxx.supabase.co

5. SUPABASE_SERVICE_ROLE_KEY
   Giá trị: Service role key (chỉ dùng server-side)

---------------------------------------------------

[SAU KHI CẤU HÌNH XONG]

1. Redeploy dự án trên Vercel (Deployments -> Redeploy).

2. CHẠY SEED DATA (Quan trọng):
   Dữ liệu cloud trống, cần seed từ local.

   B1: Mở Terminal:
       cd empulse-next
   B2: Đảm bảo `.env.local` trỏ đúng Supabase Cloud.
   B3: Chạy lệnh:
       npm run seed

   Kết quả: "✅ Seed completed successfully!"
   Sau đó đăng nhập bằng admin@empulse.com / password123.
